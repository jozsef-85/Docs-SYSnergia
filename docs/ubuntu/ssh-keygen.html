SSH — Par de Claves Ed25519

Configura autenticación por clave pública para acceso SSH seguro sin contraseña. Más seguro y cómodo que la autenticación clásica.

Ubuntu 24.04 OpenSSH 9.x Intermedio
01 Generar el par de claves en el cliente ▾

Ejecuta desde tu máquina local. Se genera un par Ed25519 — más seguro y compacto que RSA.

bash — cliente local
$ ssh-keygen -t ed25519 -C "mi-servidor-ubuntu"

Generating public/private ed25519 key pair.
Enter file: /home/user/.ssh/id_ed25519  ← Enter para aceptar
Enter passphrase:                        ← Opcional pero recomendado

Your identification has been saved in /home/user/.ssh/id_ed25519
Your public key has been saved in /home/user/.ssh/id_ed25519.pub
→ Prefiere ed25519 sobre RSA. Es más seguro, claves más cortas y compatible con OpenSSH 6.5+. Para compatibilidad máxima usa -t rsa -b 4096.
02 Copiar la clave pública al servidor ▾

Transfiere tu clave pública con ssh-copy-id. Crea ~/.ssh/authorized_keys con los permisos correctos automáticamente.

bash — cliente
$ ssh-copy-id -i ~/.ssh/id_ed25519.pub usuario@192.168.1.100

Number of key(s) added: 1
Now try: ssh 'usuario@192.168.1.100'

Si ssh-copy-id no está disponible, método manual:

bash — método manual
$ cat ~/.ssh/id_ed25519.pub | ssh usuario@servidor \
    "mkdir -p ~/.ssh && chmod 700 ~/.ssh && \
     cat >> ~/.ssh/authorized_keys && \
     chmod 600 ~/.ssh/authorized_keys"
03 Configurar sshd_config en el servidor ▾
/etc/ssh/sshd_config
PermitRootLogin                  no
PubkeyAuthentication             yes
AuthorizedKeysFile               .ssh/authorized_keys
PasswordAuthentication           no
KbdInteractiveAuthentication     no
Port                             2222
AllowUsers                       jose admin
bash — servidor Ubuntu
# Verificar sintaxis y reiniciar
# sshd -t && systemctl restart ssh
# systemctl status ssh
⚠ NO deshabilites PasswordAuthentication hasta verificar que conectas con la clave. En Ubuntu el servicio se llama ssh, no sshd.
04 Simplificar con ~/.ssh/config ▾
~/.ssh/config — cliente
Host mi-ubuntu
    HostName     192.168.1.100
    User         jose
    Port         2222
    IdentityFile ~/.ssh/id_ed25519
bash
$ ssh mi-ubuntu           # conecta directo
$ scp archivo.txt mi-ubuntu:/tmp/
